Three.js: Precision Engineering

Directing the Render Loop for Maximum Industrial Grit.

Posted by Scott Grocott on January 21, 2026

In the last leg of the shootout, we looked at how A-Frame makes scene construction incredibly fast. But for a project like Metal Throne—where dense industrial geometry and complex lighting are the stars—sometimes you need to get closer to the metal. Today, we look at Three.js.

Stripping Away the Middleman

While A-Frame is built on top of Three.js, it uses a DOM-based Entity Component System (ECS). By moving directly into Three.js, I’ve removed the overhead of the HTML layer. This allows the Metal Throne engine to talk directly to the GPU, offering a level of fine-tuning that is essential for maintaining a high frame rate in dense environments.

The Loading Logic

Using the same manifest.json from the previous tests, I wrote a dedicated loader in Three.js. Without the convenience of A-Frame components, I manually handle the GLTFLoader and the recursive positioning of each mesh.

// Three.js: Direct JSON to Scene implementation
const loader = new THREE.GLTFLoader();

fetch('assets/manifest.json')
  .then(res => res.json())
  .then(data => {
    data.hub_elements.forEach(item => {
      loader.load(item.src, (gltf) => {
        const model = gltf.scene;
        model.position.set(item.pos.x, item.pos.y, item.pos.z);
        model.rotation.set(item.rot.x, item.rot.y, item.rot.z);
        scene.add(model);
      });
    });
  });

Performance & Fine-Tuning

The real power of Three.js lies in its Render Loop. I can now control exactly how shadows are calculated, how textures are sampled, and how much "post-processing" grit (like bloom and film grain) is applied to the scene without being limited by an existing framework's presets.

In the screenshots below, notice the sharpness of the shadows on the fuel silos and the subtle metallic sheen on the pathways. By manually adjusting the renderer.shadowMap type and the anisotropy of the textures, I can squeeze every ounce of visual fidelity out of the browser.
