A-Frame: The Visual Architect

Building the Fuel Yard with JSON and the Visual Inspector.

Posted by Scott Grocott on January 20, 2026

As the "Great Engine Shootout" continues, A-Frame has emerged as a frontrunner for rapid iteration. While Three.js offers raw power and Babylon.js brings industrial-grade features, A-Frame offers something neither can match as easily: a built-in, real-time visual 3D inspector that works right in the browser.

Dynamic JSON Instantiation

In the Metal Throne ecosystem, the scene doesn't live in the HTML. It lives in the manifest.json. I wrote a custom A-Frame component that acts as our "Scene Builder." It fetches our coordinates and injects <a-entity> tags into the DOM on the fly.

// Scene Loader Component
AFRAME.registerComponent('json-model-loader', {
  init: function () {
    fetch('assets/manifest.json')
      .then(res => res.json())
      .then(data => {
        data.hub_elements.forEach(item => {
          let el = document.createElement('a-entity');
          el.setAttribute('gltf-model', item.src);
          el.setAttribute('position', item.pos);
          el.setAttribute('rotation', item.rot);
          this.el.appendChild(el);
        });
      });
  }
});

The Secret Weapon: The Inspector

One of the most powerful features of A-Frame is the Visual Inspector. By hitting Ctrl + Alt + I on the keyboard, I can freeze the live scene and jump into a 3D editor mode.

This is where the "0,0,0 workflow" really pays off. I can grab a fuel silo, nudge it five centimeters to the left, or rotate a pathway to better align with the terrain texture, all without touching a line of code. Once it's perfect, I can simply copy the new coordinates and update the JSON file.
