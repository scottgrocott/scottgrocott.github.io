# Metal Throne Engine — Session 3 Starter Prompt

## Project Summary
Browser-based game engine: BabylonJS + Rapier3D + YUKA + ToneJS. ~45 ES modules, no bundler, vanilla JS static files. 5 game configs. Built-in localhost editor (F key = freecam + editor panel).

---

## Current State: Working ✅

- Engine boots, blue sky, directional sun light (sky.js)
- Player spawns ON terrain surface with Rapier capsule physics
- WASD + mouse look, freecam (F), shooting (left click)
- Heightmap terrain: PNG loaded via editor → vertex-stamped → Rapier HeightField collider
- Terrain visible, correct orientation, player walks on it with physics
- Scatter sprites (vegetation/rocks) placed on terrain surface
- All 5 configs load and switch cleanly
- Enemies spawn and move: cars ram player, forklifts chase, drones patrol above terrain
- Enemy hit detection + kill/respawn cycle works
- Shelter drop system works (F → editor → drop shelter)
- Audio works, config switching works
- Minimap shows player position + enemy dots

## Architecture: Module Registry Pattern
All enemy consumers (hud.js, minimap.js, basicGun.js, editor.js) import from `enemyRegistry.js` NOT `enemyBase.js`:
- `getEnemies()` — all instances
- `getEnemyCount()` — alive count  
- `hitEnemy(enemy, damage)` — apply damage
- `clearEnemies()` — full teardown on config switch

## Key Architecture Decisions & Pitfalls

1. **RAPIER is ESM** in `physics.js` — `window.RAPIER = RAPIER_MODULE` makes it global
2. **YUKA is classic script tag** — all usage guarded with `if (!window.YUKA) return`
3. **BabylonJS Vector3 cannot be spread** — always extract: `const x = +v.x`
4. **NaN kills Rapier permanently** — every setTranslation uses `safeVec3()` from physics.js
5. **`camera.globalPosition`** not `camera.getAbsolutePosition()`
6. **Terrain heightmap** is vertex-stamped manually (not CreateGroundFromHeightMap which ignores data URLs). V-axis is flipped: `v = 1.0 - (row/subdiv)`
7. **`getTerrainHeightAt(x,z)`** in terrainMesh.js is the single source of truth for all height queries — spawn.js, flatnav.js, cars.js, forklifts.js, drones.js, scatter.js all use this
8. **Terrain physics** via Rapier HeightField collider in `addTerrainCollider()` — same pixel data, same V-flip as visual mesh
9. **`tickYUKA()`** must be called every frame in main.js render loop
10. **`setPlayerRigRef(playerRig)`** must be called after initPlayer() so enemy distToPlayer works
