## index.html Pattern (for all game deployments)
```html
<!-- Auto-detect engine: local dev vs GitHub Pages — only ONE script loads -->
<script>
  const engineSrc = (location.hostname === 'localhost' || location.hostname === '127.0.0.1')
    ? 'http://localhost:83/mt-engine/main.js'
    : 'https://scottgrocott.github.io/mt-engine/main.js';
  const s = document.createElement('script');
  s.type = 'module';
  s.src = engineSrc;
  document.head.appendChild(s);
</script>
```
Do NOT use a plain `<script type="module" src="main.js">` — that breaks when game and engine are in different directories.

## Architecture Reminder

### ENGINE_ROOT & Path Resolution
- `ENGINE_ROOT` = derived from `main.js` script src URL via `new URL(script.src).href`
- `_pageBase` = `window.location.origin + pathname` (handles trailing slashes correctly)
- Level files fetched relative to `_pageBase` (the game directory)
- Asset paths in config prefixed with `ENGINE_ROOT` if relative

               optional chaining on all getElementById, meta title updates page+loading screen,

## Deployment Workflow
1. Edit config in engine editor (localhost:83/mt-engine)
2. Set GitHub Pages engine URL in editor DEPLOY section (one-time)
3. Click Export Game → downloads index.html + game-config.json
4. Drop files into mt-games/[game-folder]/
5. Add level-N.json files for each level
6. Push to GitHub → live at scottgrocott.github.io/mt-games/[game-folder]/?level=N
